# 摸鱼宝 H5 产品规格文档

## 1. Concept & Vision

**摸鱼宝** —— 职场人的碎片时间情绪代偿工具

核心理念：让摸鱼从"偷偷摸"变成"合法喘息"。不是让用户沉迷游戏，而是让辛苦一天的打工人，在碎片时间里获得3秒钟的快乐和共鸣，然后继续面对明天的苟且。

产品调性：温暖、幽默、有共鸣。视觉上要有"职场黑色幽默"的感觉，不是丧，是苦中作乐。

目标用户画像：工作1-5年的职场人，互联网/金融/制造等行业，加班常态化，对"内卷"有强烈共鸣，疲惫但还在坚持。

## 2. Design Language

### 美学方向
职场版的"深夜食堂" —— 暖色调为主，让人感觉在疲惫的一天后被温柔接住。不是年轻人的潮酷，是"打工人专属的疗愈角落"。

### 色彩系统
```
主色：#FF6B35（温暖橙红）— 活力、温暖
辅色：#2D3142（深藏青）— 沉稳、职场感
强调色：#FFD166（金黄）— 成就、奖励
背景色：#F8F5F2（暖白）— 干净、轻松
文字色：#2D3142（主）/ #8D99AE（次）
成功色：#06D6A0（薄荷绿）
危险色：#EF476F（珊瑚红）
```

### 字体
- 主字体：系统默认中文字体（苹方/思源黑体/Noto Sans CJK）
- 数字/英文：DIN Alternate / SF Pro Display
- 标题：加粗大号，正文：常规

### 动效哲学
- 进入动画：轻微弹性，如"气泡上浮"般自然
- 按钮反馈：按下有"凹陷"感
- 成就解锁：轻微震动 + 金光扩散
- 过渡：200-400ms，ease-out为主
- 禁止：过于复杂/花哨的动画，违背"轻松"调性

### 视觉资产
- 图标：Lucide Icons（线性风格）
- 插画：扁平职场场景插画（可后期添加）
- 装饰：几何图形（圆、矩形）组合，模拟便利贴/便签效果

## 3. Layout & Structure

### 页面结构
```
┌─────────────────────────────────────┐
│  Header（固定）                      │
│  - Logo + 今日许可证入口             │
│  - 用户头像/成就徽章入口             │
├─────────────────────────────────────┤
│                                     │
│  主内容区（滚动）                    │
│  - 核心功能模块（卡片式）            │
│  - 各模块独立但视觉统一              │
│                                     │
├─────────────────────────────────────┤
│  Bottom Tab（固定）                  │
│  [首页] [劫后余生] [搭子] [我的]    │
└─────────────────────────────────────┘
```

### 页面列表
1. **首页**：今日许可证 + 快速入口 + 今日提示
2. **劫后余生**：记录入口 + 今日汇总 + 月度报告
3. **搭子**：职场画像 + 匹配 + 搭子互动
4. **我的**：成就徽章 + 摸鱼统计 + 设置

### 响应策略
- 移动端优先（375px-428px）
- 最大宽度480px居中
- 单列布局，避免复杂网格

## 4. Features & Interactions

### 4.1 今日许可证（首页核心）

**功能**：
- 每日零点自动生成
- 展示：允许摸鱼分钟数 / 已使用分钟数 / 剩余分钟数
- 进度条可视化
- 会员双倍配额标识

**交互**：
- 首次进入：弹窗展示今日许可证，带轻微弹性动画
- 每日首次进入首页：卡片顶部高亮
- 配额用完：进度条变红，显示"明天再来"

**状态**：
- 正常：橙色调
- 即将用完（<5分钟）：黄色警示
- 已用完：灰色

### 4.2 劫后余生（核心模块）

**功能**：
- 记录"差点发生的坏事"
- 内置事件类型：差点加班、差点开会、差点被骂、差点背锅、差点迟到、差点做汇报、差点被拉群
- 自由描述输入
- 今日汇总展示
- 月度报告生成

**交互流程**：
1. 点击"记录差点" → 选择事件类型（卡片式）
2. 选择后 → 弹出文本框"发生了什么？"
3. 提交 → 显示"这个劫你逃掉了！" + 本月累计
4. 若今日记录≥3 → 解锁"劫后英雄"成就提示

**差一点钩子**：
- 每月最后3天：显示"本月还差X天满勤"
- 连续记录3天以上：显示"连续X天记录，离'小有身手'还差Y天"

**状态**：
- 空状态：插画 + "今天没有差点？说明你今天很幸运！"
- 记录中：输入框 + 类型选择
- 已记录：汇总卡片 + 历史列表

### 4.3 职场同类搭子

**功能**：
- 职场画像录入（5道选择题）
- 匹配候选人展示（匿名标签）
- 搭子配对确认（双向确认）
- 搭子早晚报
- 掩护请求（紧急呼叫）
- 搭子续约/解散

**画像维度**：
1. 行业（互联网/金融/制造/教育/其他）
2. 职级（P5及以下/P6/P7/M1+/实习）
3. 加班强度（从不/偶尔/经常/天天）
4. 摸鱼风格（低调型/奔放型/间歇型）
5. 工龄（0-1年/1-3年/3-5年/5-10年/10年+）

**交互**：
- 未画像：引导问卷（一步步填写）
- 已画像未配对：推荐候选人卡片
- 已配对：搭子信息卡片 + 操作按钮
- 掩护请求：紧急红色弹窗 + 对方即时通知

**状态**：
- 未填写画像：引导页
- 画像完成未配对：匹配页
- 配对中（待确认）：等待对方确认
- 已配对：搭子主页

### 4.4 成就系统

**徽章列表**：
| 徽章 | 条件 | 图标 |
|------|------|------|
| 小试牛刀 | 完成职场画像 | 🎯 |
| 小有身手 | 逃过3劫 | 🧘 |
| 金钟罩 | 逃过10劫 | 🛡️ |
| 铁布衫 | 逃过30劫 | ⚙️ |
| 灵魂伴侣 | 和搭子连续互动7天 | 💑 |
| 摸鱼老手 | 累计使用7天 | 🐟 |
| 全勤摸鱼 | 连续30天每天使用 | 🏆 |
| 劫后英雄 | 单日记录3个劫 | 🦸 |

**交互**：
- 成就列表：已获得 + 未获得（显示解锁条件）
- 解锁瞬间：弹窗 + 金光动画 + 分享按钮
- 进度展示：环形进度条

### 4.5 定时推送（H5限制说明）

**H5局限性**：无法主动推送通知，需用户打开页面

**解决方案**：
- 首页固定显示"上次回来时间"
- 搭子功能：搭子在线时可即时消息
- 用户可设置"提醒时间"，到时间时浏览器通知（需用户授权）

## 5. Component Inventory

### 5.1 许可证卡片
- 默认：橙色边框，进度条填充
- 即将用完：黄色边框，进度条黄色
- 已用完：灰色边框，虚线进度条
- 会员：双倍徽章标识

### 5.2 事件类型选择卡
- 默认：白色背景，浅灰边框
- 选中：橙色边框，浅橙背景，轻微放大
- 点击反馈：缩放0.95 → 1.0

### 5.3 搭子卡片
- 候选人卡片：头像区 + 匿名标签 + 匹配度分
- 配对成功卡片：搭子信息 + 续约按钮 + 掩护按钮
- 等待确认卡片：倒计时 + 取消按钮

### 5.4 成就徽章
- 已获得：彩色 + 发光边框
- 未获得：灰色 + 问号遮罩
- 新解锁：金色 + 弹跳动画

### 5.5 按钮
- 主按钮：橙色背景，白色文字，圆角12px
- 次按钮：白色背景，橙色边框，橙色文字
- 危险按钮：珊瑚红背景，白色文字
- 禁用状态：灰色，不可点击

### 5.6 输入框
- 默认：浅灰边框，暖白背景
- 聚焦：橙色边框，轻微阴影
- 错误：珊瑚红边框，错误提示文字

## 6. Technical Approach

### 前端
- 框架：Vue3 + Vite
- 样式：CSS Variables + Scoped CSS
- 状态：Pinia（轻量状态管理）
- HTTP：Axios
- 部署：Vercel / 任意静态托管

### 后端
- 框架：Python FastAPI
- 数据库：SQLite（开发）/ PostgreSQL（生产）
- 认证：UUID Token（简单够用）
- 定时任务：APScheduler

### 数据模型
```python
# 用户
User:
  - id: UUID
  - created_at: datetime
  - last_active: datetime
  - is_premium: bool
  
# 许可证
License:
  - user_id: FK
  - date: date
  - quota_minutes: int
  - used_minutes: int
  
# 劫后余生记录
Incident:
  - id: UUID
  - user_id: FK
  - date: date
  - type: str
  - description: text
  - escaped: bool
  - escaped_time: time
  
# 职场画像
WorkProfile:
  - user_id: FK
  - industry: str
  - level: str
  - overtime: str
  - style: str
  - work_years: int
  
# 搭子关系
Partner:
  - user_id: FK
  - partner_id: FK
  - matched_date: date
  - expires_date: date
  - confirmed: bool
  
# 成就
Achievement:
  - user_id: FK
  - achievement_id: str
  - unlocked_at: datetime
```

### API设计
```
GET  /api/user              - 获取用户信息（含许可证）
GET  /api/license            - 获取今日许可证
POST /api/license/reset      - 消耗配额

GET  /api/incidents          - 获取劫后余生列表
POST /api/incidents          - 记录新事件
GET  /api/incidents/summary  - 获取今日/月度汇总

GET  /api/profile            - 获取职场画像
POST /api/profile            - 提交/更新画像
GET  /api/partners/match     - 获取匹配候选人
POST /api/partners/confirm   - 确认配对
POST /api/partners/cover     - 发起掩护请求
GET  /api/partners/status    - 获取搭子状态

GET  /api/achievements       - 获取成就列表
GET  /api/achievements/stats - 获取统计数据
```
